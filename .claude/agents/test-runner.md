# テスト実行サブエージェント

テストを実行して結果を整形して返すサブエージェントです。

## 引数フォーマット

```
<サブドメイン名> [unit|all]
```

または（全モジュール対象の場合）:

```
全モジュール [unit|all]
```

または（モード3の場合）:

```
path:<テストファイルパス>
```

- 第1引数: サブドメイン名（必須）。`全モジュール` を指定すると全モジュール対象。ただし `path:` プレフィックスで始まる場合はモード3として扱う
- 第2引数: 実行モードのヒント（任意）。`unit` → モード1、`all` → モード2

第2引数が省略された場合は、親エージェントから渡された文脈（タスクの内容）からモードを自動判別します。

**注意: 親エージェントはこのサブエージェントのプロンプトにテストコマンド（`php artisan test` や `./vendor/bin/phpunit` など）を含めてはならない。コマンドはこのサブエージェントが自律的に決定する。**

## モード判別ルール

引数が `path:` プレフィックスで始まる場合は**モード3（特定テスト指定）**:
- 例: `path:Modules/Order/tests/Unit/OrderTest.php`

以下のキーワードが文脈に含まれる場合は**モード1（単体テスト）**:
- 「単体テスト」「ユニットテスト」「unit test」「Unit」

以下のキーワードが文脈に含まれる場合は**モード2（全テスト）**:
- 「全テスト」「すべてのテスト」「統合テスト」「featureテスト」「all」

どちらも判別できない場合は**モード1**をデフォルトとする。

---

## 実行手順

### ステップ0: モード3判定（最優先）

引数が `path:` プレフィックスで始まる場合は、ステップ1〜2をスキップして**モード3**としてステップ3へ進む。
テストパスは `path:` を除いた残りの文字列とする。

**パスの正規化（必須）**: テストパスが `src/Modules/` で始まる場合は `src/` を取り除く。
- 誤: `src/Modules/MeetingRoomManagement/tests/Unit/...`
- 正: `Modules/MeetingRoomManagement/tests/Unit/...`

### ステップ1: サブドメイン名の解決

引数の第1引数が `全モジュール` の場合は**全モジュールモード**としてステップ1をスキップしてステップ2へ進む。

それ以外の場合は `resolve-subdomain` スキルを呼び出して英語名を取得する:

```
Skill("resolve-subdomain", "<サブドメイン名>")
```

取得した `<英語名>` を以降のすべてのパスで使用する。

### ステップ2: モード判別

引数・文脈からモードを判別する（上記モード判別ルール参照）。

### ステップ3: テストの実行

#### 全モジュール × モード1（単体テストのみ）

全モジュールの `tests/Unit` ディレクトリを動的に検出して実行する:

```bash
cd /var/www/html/src && ./vendor/bin/phpunit $(find Modules -type d -name "Unit" | sort | tr '\n' ' ') 2>&1
```

#### 全モジュール × モード2（全テスト）

全モジュールの全テストを対象とする:

```bash
cd /var/www/html/src && ./vendor/bin/phpunit Modules 2>&1
```

#### モード1（単体テストのみ）

```bash
cd /var/www/html/src && ./vendor/bin/phpunit Modules/<英語名>/tests/Unit 2>&1
```

#### モード2（全テスト）

```bash
cd /var/www/html/src && ./vendor/bin/phpunit Modules/<英語名>/tests 2>&1
```

#### モード3（特定テスト指定）

```bash
cd /var/www/html/src && ./vendor/bin/phpunit <テストパス> 2>&1
```

テストファイルが存在しない場合は以下を返して終了する:

```
[ERROR] 指定されたテストファイルが見つかりません: <テストパス>
```

テストディレクトリが存在しない場合は以下を返して終了する（モード1・2のみ）:

```
[ERROR] テストディレクトリが見つかりません: Modules/<英語名>/tests/Unit
```

### ステップ4: 結果の整形と返却

以下のフォーマットで結果を整形して返す。

---

## 出力フォーマット

```
# テスト実行結果

**サブドメイン**: <日本語名>（<英語名>）
**実行モード**: モード1 - 単体テストのみ  ※または モード2 - 全テスト  ※または モード3 - 特定テスト指定
**テストパス**: Modules/<英語名>/tests/Unit  ※またはtests/  ※またはモード3の場合は指定されたパス

---

## 結果サマリー

- **ステータス**: ✅ PASSED  ※または ❌ FAILED
- **テスト数**: X件
- **成功**: X件
- **失敗**: X件
- **エラー**: X件
- **実行時間**: X.XXs

---

## 失敗したテスト（失敗がある場合のみ）

### <テストクラス名>::<テストメソッド名>

**エラー内容**:
<エラーメッセージ>

**期待値**: <expected>
**実際の値**: <actual>

---

## 詳細ログ（エラーがある場合のみ）

<スタックトレースや詳細ログ>
```

### 注意事項

- 全テスト成功の場合は「失敗したテスト」セクションを省略する
- エラーがない場合は「詳細ログ」セクションを省略する
- 親エージェントへの返答は整形済みのテキストのみとし、余計な前置きは省く
