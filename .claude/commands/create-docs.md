下記の手順で、引数として渡されたサブドメイン（$ARGUMENTS）のドキュメントを新規作成してください。

サブドメイン: $ARGUMENTS

## 手順

以下のステップを順番に実行してください。各ステップは前のステップの成果物を参照します。
**各ステップの完了後、必ずレビューを求めてから次のステップへ進むこと。**

---

### ステップ1: PRD作成
`prd-writing`スキルを使用して、サブドメイン「$ARGUMENTS」のプロダクト要求定義書（PRD）を作成する。

**【レビュー】ステップ1完了後、以下の選択肢をユーザーに提示して入力を待つ:**

```
✅ ステップ1「PRD作成」が完了しました。

次のアクションを選択してください:

1. 次のステップへ進む（ステップ2: ユースケース図作成）
2. このステップの成果物を修正・改善する　→ 改善内容を入力してください
3. 内容について質問する　→ 質問を入力してください
4. このステップをやり直す（最初からやり直し）
5. 作業を中止する
```

- **1** を選択 → ステップ2へ進む
- **2** を選択 → ユーザーの改善指示を受け取り修正後、再度このレビューを提示する
- **3** を選択 → ユーザーの質問に回答後、再度このレビューを提示する
- **4** を選択 → ステップ1を最初からやり直し、完了後に再度このレビューを提示する
- **5** を選択 → 作業を終了する
