# プロダクト要求定義書 (Product Requirements Document)
# 予約管理サブドメイン（ReservationManagement）

## プロダクト概要

### 名称
**RoomBooker 予約管理システム (ReservationManagement)** - ダブルブッキングをゼロにする会議室予約サービス

### プロダクトコンセプト
- **迷わない空き状況確認**: 会議室と時間帯の空き状況を一覧で即座に把握できる（個人情報は一切表示しない）
- **重ならない予約保証**: 10分バッファタイムの自動挿入とDBトランザクションによりダブルブッキングを物理的に防止する
- **隠す個人情報保護**: 予約一覧には会議室名と時間帯のみを公開し、担当者名・連絡先は予約者本人のみが確認できる

### プロダクトビジョン
RoomBooker予約管理システムは、社内会議室の予約トラブルをゼロにすることを最優先にした予約サービスである。
利用者は移動中でもスマートフォンから30秒以内に予約を完了でき、空き部屋を探し回る時間を本来の業務に充てられる。
10分バッファタイムの強制挿入により「前の人がまだいる」という物理的衝突を根絶し、個人情報保護の設計によりプライバシーを守りながら透明性の高い空き状況を全社に提供する。
予約は当日から14日先まで受け付け、時間帯は最短30分・最長4時間で柔軟に設定できる。

### 目的
- 会議室の二重予約・物理的衝突を完全に防止する
- 利用者が空き状況をリアルタイムで素早く確認できるようにする
- 担当者の個人情報（連絡先）を一覧から保護する
- 会議室予約の登録・変更・キャンセルを利用者が直接操作できるようにする

## ターゲットユーザー

### プライマリーペルソナ: 佐藤さん（35歳、プロジェクトマネージャー）

- 複数プロジェクトを並行して担当しており、スケジュール調整が日常的に発生する
- 社内チャットとスマートフォンを主な業務ツールとして使用
- システムへの習熟度は一般的（Webフォームの操作は問題なく行える）
- **現在の課題**: 空き部屋を探してフロアを歩き回ったり、チャットで他のチームに空き確認をするのが時間の無駄。会議直前に他の予約と重なって揉めることもある
- **期待する解決策**: 移動中にスマートフォンから30秒以内に予約を終わらせたい。前の会議が長引いても次の予約者と衝突しない仕組みが欲しい

## 成功指標（KPI）

### プライマリーKPI
- **ダブルブッキング発生件数**: 0件（リリース後6ヶ月間を通じて維持）
- **MAU（月間アクティブユーザー）**: 導入3ヶ月で社員の80%以上が月1回以上予約操作を実行

### セカンダリーKPI
- **予約完了時間**: 操作開始から予約確定まで平均45秒以内（ユーザビリティテストで計測）
- **特定会議室への予約集中率**: 最も人気の会議室への予約集中を20%改善（リリース前比）
- **個人情報漏洩インシデント**: 0件（連絡先がAPIレスポンスに含まれる事故を防止）

## 機能要件

### コア機能（MVP）

#### 1. 会議室の予約登録

**ユーザーストーリー**:
利用者として、希望の会議室と時間帯を確保するために、会議室の予約を登録したい。

**受け入れ条件**:
- [ ] 予約名（ReservationName）を1文字以上50文字以内で入力できる
- [ ] 担当者名（ContactPersonName）を1文字以上30文字以内で入力できる
- [ ] 連絡先（ContactEmail）を有効なメールアドレス形式で入力できる
- [ ] 予約時間帯（ReservationTimeRange）を開始・終了で指定でき、8:00〜22:00の範囲内で設定できる
- [ ] 予約時間帯は最短30分・最長4時間の制約が守られており、違反時はエラーメッセージが表示される
- [ ] 予約対象日は当日〜14日先（ReservablePeriod）の範囲のみ選択できる
- [ ] 前の予約終了時刻から10分（BufferTime）以内に開始する予約は登録できず、エラーメッセージが表示される
- [ ] 同一会議室・同一時間帯への二重予約は、同時操作を含めDBレベルで防止される
- [ ] 必須項目が未入力の場合、バリデーションエラーメッセージが日本語で表示される
- [ ] 予約登録が成功した場合、予約完了画面が表示される
- [ ] 登録完了後、担当者の連絡先（ContactEmail）に予約確認メールが送信される

**優先度**: P0（必須）

---

#### 2. 空き状況の確認（予約一覧）

**ユーザーストーリー**:
利用者として、どの会議室がいつ空いているかを素早く把握するために、予約一覧（ReservationList）を確認したい。

**サブドメイン連携**:
本ユースケースでは、会議室に関する情報を **MeetingRoomManagement（会議室管理）サブドメイン** から取得する。
取得する情報は以下の通り:
- 会議室ID（MeetingRoomId）
- 会議室名（MeetingRoomName）
- 定員（Capacity）
- 備品（Equipment）

**受け入れ条件**:
- [ ] 全会議室の予約状況を一覧で表示できる（会議室名・予約時間帯のみ表示）
- [ ] 担当者名・連絡先（ContactEmail）は一覧に表示されない
- [ ] 日付でフィルタリングでき、当日〜14日先の範囲で選択できる
- [ ] 会議室でフィルタリングできる
- [ ] フィルタリング結果が0件の場合、「該当する予約はありません」と表示される
- [ ] 予約件数が20件を超える場合、ページネーションが適用される
- [ ] 一覧表示は1秒以内にレンダリングが完了する

**優先度**: P0（必須）

---

#### 3. 予約のキャンセル

**ユーザーストーリー**:
利用者として、不要になった予約を解放して他の人が使えるようにするために、予約をキャンセルしたい。

**受け入れ条件**:
- [ ] 予約者本人のみキャンセル操作が可能である
- [ ] キャンセル前に確認ダイアログが表示される
- [ ] キャンセルが完了すると即座に空き状況に反映され、該当時間帯が他の利用者に予約可能になる
- [ ] キャンセル完了後、担当者の連絡先（ContactEmail）にキャンセル通知メールが送信される

**優先度**: P0（必須）

---

#### 4. 予約の変更

**ユーザーストーリー**:
利用者として、予約内容を修正する必要が生じた際に、予約時間帯や予約名を変更したい。

**受け入れ条件**:
- [ ] 予約者本人のみ変更操作が可能である
- [ ] 変更フォームには変更前の値が初期表示される
- [ ] 変更後の時間帯が登録時と同様のバリデーション（ReservablePeriod・ReservationTimeRange・BufferTime・二重予約防止）を通過した場合のみ変更が確定する
- [ ] 変更完了後、担当者の連絡先（ContactEmail）に変更通知メールが送信される

**優先度**: P0（必須）

---

### 将来的な機能（Post-MVP）

#### 5. 予約のリマインダー通知

予約開始の15分前に担当者へメール通知を送信する機能。
会議の準備忘れ・前の会議への退室促進にも活用できる。

**優先度**: P1（重要）

---

#### 6. 利用履歴・統計レポート

管理者向けに会議室ごとの予約件数・稼働率を月次・週次で集計して表示するレポート機能。
特定の会議室への偏りを可視化し、会議室の配分改善に活用する。

**優先度**: P2（できれば）

## 非機能要件

### パフォーマンス
- 予約一覧表示: 100件まで1秒以内にレンダリング完了
- 予約登録・変更・キャンセル: 操作完了まで2秒以内

### ユーザビリティ
- 初回利用者が操作開始から予約完了まで平均45秒以内に完了できる（マニュアルなしで）
- エラーメッセージは原因と修正方法を日本語で明示する
- PCブラウザ（Chrome/Firefox/Edge 最新版）およびスマートフォンブラウザで正常に動作すること

### 信頼性
- ダブルブッキング: DBトランザクションと楽観的ロックにより0件を保証する
- 登録・変更・キャンセル操作は原子性を保証し、途中失敗時はロールバックする
- 稼働率: 99%以上（月次計測）、障害発生時は1時間以内に復旧する

### セキュリティ
- 担当者の連絡先（ContactEmail）はAPIレスポンス・一覧画面に一切含めない
- 予約の変更・キャンセルは予約者本人のみ実行可能（認証・認可を必須とする）
- 予約の登録・閲覧は認証済みの社員ロールのユーザーのみ実行可能

### スケーラビリティ
- 予約件数: 最大10,000件まで正常動作すること
- 会議室数の増加（MeetingRoomManagementサブドメイン側の変更）に対して、MeetingRoomIdを介した疎結合設計により影響を最小化すること

---

## スコープ外

- スマートロック連携（物理キーの解錠・施錠）
- 可動式備品（ポータブルプロジェクターなど）の在庫管理
- 会議室の座席レイアウト・フロアマップ表示
- 外部カレンダー（Google Calendar・Outlook等）との同期
- 会議室の登録・編集・削除（会議室管理サブドメインのスコープ）
