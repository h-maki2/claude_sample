# RoomBooker要件定義メモ (雑記) - V2

## 📌 プロダクトの方向性
* **名前**: RoomBooker（仮）
* **一言で**: 調整コストをゼロにする会議室管理システム
* **狙いたいこと**:
    * 「迷わない」：空き状況と備品をパッと見でわかるように。
    * 「重ならない」：**10分バッファ**を自動挿入。物理的にダブルブッキングを潰す。
    * 「隠す」：予約は見せるけど、連絡先とかの個人情報は適切にガード。
* **ビジョン**: 会議室予約のトラブルをゼロにして、本来の業務に即集中できる環境を作る。

## 👤 ターゲット（ペルソナ）
* **佐藤さん (35歳 PM)**: 複数プロジェクトを回してて超忙しい。
* **悩み**: 空き部屋探しでフロアを歩き回ったり、チャットで調整するのが時間の無駄。
* **理想**: 移動中にスマホでサクッと、**30秒以内**に予約を終わらせたい。

## 🎯 目標 (KPI)
* **マスト**:
    * MAU：導入3ヶ月で社員の80%以上。
    * **ダブルブッキング発生：絶対0件。**
* **できれば**:
    * 予約完了まで45秒以内。
    * 特定の部屋への偏りを20%改善。

## 🛠️ 機能要件 (MVP) - ユースケース単位
コア機能（P0）を実際の利用シーンで整理。

### 【UC-01】会議室を予約する
* **やること**: 
    * 今日〜14日先までの空き枠を探して、30分〜最大4時間で予約。
    * 予約名、担当者名、メアドを入力。
* **システムの裏側（キモ）**:
    * **10分バッファ**: 前の予約が終わった直後10分間は、システムが自動で「準備時間」としてロック。連続予約による「前の人が退かない」問題を物理的に防ぐ。
    * **二重予約防止**: 同時操作されてもDBで厳格にチェック。

### 【UC-02】空き状況をパッと確認する
* **やること**: 
    * 一覧画面で「今、どこが空いてるか」を秒で確認。
* **プライバシー配慮**: 
    * 一覧には「会議名」と「時間」だけ表示。**担当者のメアドは絶対に出さない**。

### 【UC-03】予約をキャンセルする
* **やること**: 
    * 即座に枠が解放され、他の人が使えるようになる。


## 🛡️ 非機能（システムの中身とか）
* **信頼性**: 稼働率99%目標。止まったら1時間以内に直す。
* **整合性**: DBのトランザクションで二重予約をガチガチに弾く。
* **構成**: Laravel (API) + MySQL。
    * **モジュラモノリス**: 最初からドメインごとにディレクトリを分けて、将来の拡張に備える。
