# 会議室管理（カタログ機能）のアイデアメモ

## 📌 このサブドメインの役割
* **ざっくり言うと**: 会議室の「カタログ」機能。
* **役割**: 会社にどんな部屋があるのかをまとめて管理する。予約システムが「どの部屋を予約するか」を参照するためのマスターデータ。
* **知りたいこと**: 
    * どんな部屋がある？（名前、定員）
    * 設備は何がある？（ホワイトボード、プロジェクターなど）

## 🛠️ メインでやりたいこと (MVP)

### 1. 会議室情報のメンテナンス（登録・編集・削除）
* **やりたいこと**: 部屋を増やしたり、名前を変えたりできるようにする。
* **細かいルール（案）**:
    * 名前：50文字以内。
    * 定員：1〜50名で設定。
    * 備品：ホワイトボード、プロジェクター、モニターなどをポチポチ選べるように。
* **安全策**: **「予約が入っている部屋」は消せないようにする。** 間違えて消して予約が迷子になるのを防ぐ（警告を出してブロック）。

### 2. 会議室の一覧表示
* **やりたいこと**: 管理者が「今、うちにはどんな部屋があるか」をパッと見れるようにする

## 🚀 あとでやりたいこと (Post-MVP)
* **稼働率レポート**: どの部屋が人気か、全然使われていない部屋はどこか、を可視化して改善に役立てる。
* **備品マスタ管理**: 「新しい備品（例：Webカメラ）」が増えたときに、システム側で項目を自由に追加できるようにしたい。

## 🛡️ システム的な注意点（非機能）
* **データの整合性**: 予約データと矛盾が出ないように（予約中の削除禁止など）。
* **拡張性**: 備品の種類が将来増えてもいいように、ガチガチに固めすぎない設計にする（マスタテーブル化など）。

## ❌ 今回はやらないこと（スコープ外）
* 鍵の管理（スマートロックとか）。
* 持ち運びできる備品（ポータブルスクリーンなど）の貸出管理。
* 「机をコ字型にする」といったレイアウトの管理。
